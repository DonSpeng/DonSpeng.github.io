在GitHub上搭建个人博客

整体思想：

1.GitHub作为一个托管平台，可以托管相关文件，自然也就可以托管网页；GitHub可以生成一个GitHub Page，作为浏览

2.Hexo可以生成静态的html页面，采用MarkDown的标记方式，用Node.js生成的

3.GitHub和Hexo两个工具，分别使用了git命令和node.js，所以需要安装相关工具（一般默认安装就可以，一路next）

准备工作：

1.注册github账号

2.Node.js

3.安装Hexo

4.安装git

验证相关软件和配置是否完成：

cmd命令行如下：

1.git –version

2.node -v

3.npm -v

当验证命令都提示安装成功后，再进行下面的操作步骤一步一步来

一、GitHub生成GitHub Pages页面

1.配置相关ssh

一路Enter过来就好，待秘钥生成完毕，会得到两个文件id_rsa和id_rsa.pub，打开id_rsa.pub，复制里面的所有内容（原封不动的复制），然后进入github-setting-ssh配置，添加ok！

测试是否成功：

​ ssh -T git@github.com，如果提示如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，看到success即可！

配置用户名和密码：

​ $ git config –global user.name “liuxianan”// 你的github用户名，非昵称
​ $ git config –global user.email “xxx@qq.com”// 填写你的github注册邮箱

2.建立仓库

​ 仓库的名字为：你的github用户名.github.io,比如你的用户名为she，那么，新建的仓库的名字为she.github.io 至此，github的账户的page配置完成

二、Hexo配置自己的页面

1.Hexo——安装

1…建立一个文件夹，专门用来存放Hexo的相关文件和主题，以及Blog的。

2…进入文件夹目录，此处为Blog（命令行cd 空格 文件夹路径（退回上一级目录cd ..）），执行下面的命令

​ npm install -g hexo

​ hexo init

​ 至此，初始化工作完成

如果想查看默认的Hexo的文件，可以继续下面的命令

​ hexo g

​ hexo s

浏览器进入http://localhost:40000,可以查看默认网页

2.Hexo——主题

​ 当然默认主题肯定不能满足需求，可以去Hexo的官网上找一个自己喜欢的主题，网址：https://hexo.io/themes/

​ 我选用了next主题，比较简洁：https://github.com/iissnan/hexo-theme-next，可以到相关主题的界面查看相关配置，这里就以next为例进行讲解

执行下面命令（仍然在文件夹下面），该命令就是从github上把主题clone下来，放到themes/next文件夹下面

​ git clonehttps://github.com/iissnan/hexo-theme-next themes/next

3.Hexo——配置

​ 在Blog文件夹下面，找到——config.yml文件，记事本打开

​ 注意：文件命令，：后面都要有一个空格

​ 配置theme：next（名字为你下载的主题的本地文件夹的名字）

​ 配置deploy：

​ deploy:
type: git
repository: https://github.com/xxxx/xxxx.github.io.git（该地址可以从github上拷贝）
branch: master

​ 其他参数比如主题等，可以自由配置

​ 具体主题的配置查看相关主题的说明文档

4.Hexo——常用命令

​ 配置好后执行下面命令：

​ hexo clean

​ hexo g

​ hexo d

至此，整个配置流程结束，可以到自己的github主页上查看自己发布的网页https://donspeng.github.io

三、自己写文档，发布到github上去

​ Blog路径下执行下面命令：

​ hexo new ‘BlogOnGitHub’

​ 创建之后，hexo会帮我们在_posts下生成相关md文件；用markDown编辑器打开，进行编辑（熟悉使用MarkDown的语法）

​ 在Blog->Source文件夹下面执行命令：

​ hexo d

相关知识：

附录：MarkDown语法知识

标题：

​ 标题是每篇文章都需要也是最常用的格式，在 Markdown 中，如果一段文字被定义为标题，只要在这段文字前加 # 号即可。

一级标题

二级标题

三级标题

以此类推，总共六级标题，建议在井号后加一个空格，这是最标准的 Markdown 语法。

列表：

​ 熟悉 HTML 的同学肯定知道有序列表与无序列表的区别，在 Markdown 下，列表的显示只需要在文字前加上 - 或 * 即可变为无序列表，有序列表则直接在文字前加1. 2. 3. 符号要和文字之间加上一个字符的空格。
